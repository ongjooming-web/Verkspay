### Onboarding System (Final Implementation)
**Status:** ✅ Complete and deployed

#### Tour Behavior (One Chance Only)
- Skip tour → `completeOnboarding()` → sets `onboarding_completed = true`
- Complete full tour → same result
- On every page load: if `completed === true`, tour NEVER shows again
- Logic: `showTour = (completed === false && tour_step === 0)`

#### Progress Bar (Independent)
- Shows until all 5 tasks complete OR user dismisses with X
- Dismiss sets `onboarding_dismissed_at = NOW()`
- If dismissed: stays hidden even if tour running
- **Celebration state:** If all 5 tasks done while dismissed, show "All set! 🎉" for 5s then hide
- Independent from tour: skipping tour doesn't affect progress bar

#### Task Tracking (5 Tasks)
1. Business Profile: `company_name` not null/empty
2. Connect Stripe: `stripe_customer_id` exists
3. Add First Client: `clients` count ≥ 1
4. Send First Invoice: `invoices` count ≥ 1
5. Generate AI Insights: `latest_insights` not null/has content

#### Immediate State Updates
- `completeOnboarding()` → immediately sets local state (completed: true)
- `dismissProgress()` → immediately sets local state (dismissed: true)
- `updateTourStep()` → immediately sets local state (tour_step: value)
- No waiting for page reload — instant UI feedback

#### Frontend Components
- **OnboardingTour:** Renders only if `!status.completed && tour_step === 0`
- **OnboardingProgress:** Hides if `completed` OR (`dismissed` AND tasks incomplete) OR all tasks done
- **useOnboarding hook:** Manages fetch, state, showTour/showProgress logic via useMemo

#### Database

  - `onboarding_completed` (BOOLEAN DEFAULT FALSE)
  - `onboarding_step` (INTEGER DEFAULT 0)
  - `onboarding_dismissed_at` (TIMESTAMPTZ DEFAULT NULL)

### Account Deletion Fix
**Status:** ✅ Deployed (commit c0a42ee)  
**Root Cause:** Frontend was missing Authorization header in DELETE request

#### The Issue
- Settings page called `/api/account/delete` WITHOUT auth token
- Backend `requireAuth()` failed because no "Bearer {token}" header
- Error: "Not authenticated"

#### The Fix
```typescript
// Before: no auth header
const response = await fetch('/api/account/delete', {
  method: 'DELETE',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: user.email })
})

// After: added auth token
const { data: { session } } = await supabase.auth.getSession()
const token = session?.access_token
const response = await fetch('/api/account/delete', {
  method: 'DELETE',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({ email: user.email })
})
```

#### Deletion Sequence (Correct)
1. Fetch session + extract token
2. DELETE request with Authorization header
3. API verifies auth with `requireAuth()`
4. Deletes: reminders_log → payment_records → payment_methods → invoices → clients → proposals → profile → auth user
5. Sends confirmation email
6. Redirects to home

#### Logging Added
- Backend: auth header check, email validation, user verification, deletion step tracking
- Frontend: response status, error details with HTTP code
