# Session: March 29, 2026 — Account Deletion Refactor (10:20–12:38 GMT+8)

## Summary
Refactored account deletion system from Next.js API route to Supabase Edge Function. Confirmed FK mapping. Ready for deployment.

## Major Work Completed

### 1. Account Deletion Architecture (Refactored)
**Moved from:** `src/app/api/account/delete/route.ts`  
**Moved to:** `supabase/functions/delete-account/index.ts`

**Why:** Service role key works properly in Edge Functions (Deno runtime has full admin access). Frontend never touches service role key.

**Flow:**
```
Frontend (JWT token) → Edge Function (service role) → Delete data + auth user → /goodbye
```

**Key Files:**
- `supabase/functions/delete-account/index.ts` — Main deletion logic
- `supabase/functions/_shared/cors.ts` — CORS headers
- `src/app/settings/account/page.tsx` — Delete confirmation UI (calls Edge Function)
- Removed: `src/app/api/account/delete/route.ts`

### 2. FK Mapping — All Tables Cascade
**Confirmed:**
- `profiles.id = auth.users.id` ✅ (PRIMARY KEY constraint)
- All user tables have `user_id` FK with `ON DELETE CASCADE` ✅

**Table List (8 tables):**
- `clients.user_id` → `auth.users(id)` ON DELETE CASCADE
- `invoices.user_id` → `auth.users(id)` ON DELETE CASCADE
- `proposals.user_id` → `auth.users(id)` ON DELETE CASCADE
- `contracts.user_id` → `auth.users(id)` ON DELETE CASCADE
- `client_notes.user_id` → `auth.users(id)` ON DELETE CASCADE
- `payment_records.user_id` → `auth.users(id)` ON DELETE CASCADE
- `webhook_config.user_id` → `auth.users(id)` ON DELETE CASCADE
- `payment_intents.user_id` → `auth.users(id)` ON DELETE CASCADE

**Implication:** Deleting `profiles` cascades to all user data. Simple, no complex deletion order needed.

### 3. Edge Function Design
**Verification Steps:**
1. Check Authorization header has Bearer token
2. Verify JWT via anon key client (extract user ID)
3. Confirm email matches request body
4. Create admin client with service role key
5. Delete profile (cascades all data)
6. Delete auth user (works now in Edge Function)
7. Return success + redirect URL to /goodbye

**Error Handling:**
- 401: Missing/invalid token
- 403: Email doesn't match auth user
- 500: Internal error (logs to Deno console)
- Soft failure on auth deletion (data already gone)
