### Architectural Decisions

1. **Currency Storage Strategy**
   - User's current preference: `profiles.currency_code` (defaults to MYR on signup)
   - Invoice's historical currency: `invoices.currency_code` (captured at creation)
   - This prevents broken displays if user changes currency later

2. **useCurrency Hook Pattern**
   - Used in authenticated pages (dashboard, invoice detail, invoice list)
   - Loads user's current preference from profiles
   - Falls back to 'MYR' on error
   - Provides `format(amount)` helper using formatCurrency()

3. **Public Payment Page Pattern**
   - Does NOT use useCurrency (no user context)
   - Reads invoice.currency_code directly from database
   - Uses formatCurrency() function directly
